# Школьная столовая - система управления питанием

 Веб-приложение для автоматизации заказа питания в школьной столовой. Система позволяет ученикам просматривать меню, формировать заказы и оплачивать их, а администраторам - управлять меню, пользователями и отслеживать заказы.

## ✨ Возможности

   Для учеников: просмотр меню, добавление блюд в корзину, оформление и оплата заказов, история заказов, личный кабинет

   Для администраторов: управление меню (добавление/редактирование/удаление блюд), управление пользователями, просмотр всех заказов

   Для персонала столовой: просмотр текущих заказов, изменение статусов готовности

## 🏗 Технологический стек

### Фронтенд

  - React 19 - библиотека для построения пользовательских интерфейсов

  - React Router DOM 7 - маршрутизация между страницами

  - Bootstrap 5 - CSS-фреймворк для стилизации компонентов

  - Vite 7 - сборщик проекта с поддержкой горячей перезагрузки

### Бэкенд

  - PocketBase - легковесный бэкенд-сервер, объединяющий базу данных SQLite, API и аутентификацию

  - PocketBase JavaScript Client 0.26 - клиент для работы с API

### Требования к окружению

Для работы с проектом необходимо:

  - Node.js версии 20.19.0 или выше (рекомендуется использовать LTS-версию)

  - npm версии 10 или выше (устанавливается вместе с Node.js)

  - Git для клонирования репозитория и управления версиями

## 🚀 Быстрый старт

### Клонирование репозитория

Выполните в терминале команду:
```
git clone https://github.com/glitch-tiger/stolovaya_serv.git
```
### Установка зависимостей фронтенда
После завершения клонирования перейдите в директорию проекта:
```
cd stolovaya_serv
```
В корневой директории проекта выполните:
```
npm install
```
Эта команда прочитает файл <b>package.json</b> и установит все необходимые зависимости, включая <b>React</b>, <b>React Router</b>,<b> Bootstrap</b>, <b>PocketBase</b> и <b>Vite</b> с плагинами. Процесс установки может занять несколько минут в зависимости от скорости интернет-соединения.

### Настройка подключения к PocketBase
Для работы приложения необходим запущенный сервер PocketBase.
    1. Скачайте PocketBase последней версии по ссылке: ``` https://pocketbase.io/docs/ ``` (выберите версию под вашу операционную систему)
    2. Распакуйте архив в удобную директорию и запустите сервер:
    
      # Windows
      pocketbase.exe serve

      # Linux/Mac
      ./pocketbase serve
      

Создайте коллекции: при первом запуске создайте коллекции ```users```, ```dishes```, ```orders```, ```carts```, ```payments```
### Настройка переменных окружения
В проекте используется файл `.env` для конфигурации подключения к PocketBase. При клонировании репозитория вы найдете файл ```.env.example``` с примером настроек:

```
CI=false
DISABLE_ESLINT_PLUGIN=true

VITE_PB_URL="https://my.pocketbase.server"
```
Чтобы настроить подключение:
   - Создайте файл ```.env ```в корне проекта (или скопируйте содержимое ```.env.example ```)
   - Отредактируйте переменную ```VITE_PB_URL```, указав URL вашего PocketBase сервера:
   - Для локальной разработки: ```VITE_PB_URL="http://127.0.0.1:8090"```


### Запуск приложения в режиме разработки

После завершения настройки проекта, в терминале пропишите:

``` npm run start ```

Vite запустит сервер разработки, и приложение станет доступно по адресу удалённого сервера. При каждом сохранении файлов страница будет автоматически перезагружаться с внесенными изменениями.

## 🛠 Быстрые команды

### `npm run start`

Запускает приложение в режиме разработки.

Страница автоматически перезагружается при внесении изменений в код.

### `npm run serve`

Предпросмотр собранного проекта.

### `npm run build`

Собирает приложение для production в папку `build`.\
Оптимизирует сборку для лучшей производительности.

Сборка минифицирована, имена файлов содержат хеши.\
Приложение готово к деплою!

### `npm run deploy`

Деплой на GitHub Pages.
